# AlgoWiki 远程部署指南

本文档面向当前这类典型场景：

- 代码仓库在本地电脑
- 服务器通过 VS Code Remote-SSH 远程维护
- 服务器位于中国内地，无法直接登录 Codex
- 服务器内存较小，不适合在远端本机构建 Docker 镜像
- 计划先做私有验证，后续切换到 RDS 和备案后的正式公网版本

## 1. 协作方式

推荐固定使用：

- 本地电脑继续使用 Codex
- VS Code 通过 `Remote-SSH` 连接服务器
- 远程窗口只负责查看文件、编辑文件、执行部署命令

不推荐在国内 ECS 上继续尝试 `codex login`。如果你在服务器上看到了 `country, region, or territory not supported` 或持续连接超时，这属于外部访问限制，不是项目配置问题。

## 2. VS Code Remote-SSH

### 2.1 SSH 配置

本地 `~/.ssh/config` 示例：

```sshconfig
Host algowiki-ecs
    HostName <your-server-ip>
    User root
    IdentityFile C:/Users/<you>/.ssh/id_ed25519
    IdentitiesOnly yes
    ServerAliveInterval 60
```

### 2.2 正确的远程目录

服务器项目目录固定为：

```text
/srv/algowiki
```

常见错误是误写成：

```text
/root/srv/algowiki
```

这不是同一个路径。

在 VS Code 中应使用：

1. `Remote-SSH: Connect to Host...`
2. 选择 `algowiki-ecs`
3. `Remote-SSH: Open Folder on Host...`
4. 输入 `/srv/algowiki`

连接成功后，远程终端执行：

```bash
pwd
ls
```

应返回：

```text
/srv/algowiki
backend  deploy  docs  frontend ...
```

## 3. 服务器基线准备

### 3.1 安装 Docker、Compose、swap

仓库内已提供脚本：

```bash
cd /srv/algowiki
chmod +x deploy/server-bootstrap.sh
SWAP_MB=4096 INSTALL_NGINX=0 CONFIGURE_UFW=1 ./deploy/server-bootstrap.sh
```

默认行为：

- 安装 Docker Engine
- 安装 Docker Compose Plugin
- 启用 Docker 开机自启
- 创建 `/swapfile`
- 开启 UFW 并仅放行 `22/80/443`

说明：

- 如果你的 ECS 只有 2G 内存，建议 `SWAP_MB=2048` 或 `4096`
- 若暂时不想动防火墙，可设 `CONFIGURE_UFW=0`
- 若准备同时安装主机 Nginx，可设 `INSTALL_NGINX=1`

### 3.2 验证服务器状态

```bash
cd /srv/algowiki
chmod +x deploy/server-verify.sh
./deploy/server-verify.sh
```

该脚本会检查：

- 系统版本
- 内存 / 磁盘 / swap
- Docker 与 Compose
- Docker 服务状态
- 当前监听端口

## 4. 本地构建与上传

### 4.1 构建并导出生产镜像

本地 Windows PowerShell：

```powershell
cd <project-root>
powershell -ExecutionPolicy Bypass -File .\scripts\build_server_image.ps1 -Tag 20260312
```

如果本地 Docker Desktop 无法访问 Docker Hub：

- 先单独测试 `docker pull node:22-bookworm-slim` 和 `docker pull python:3.12-bookworm`
- 若你已经配置了可用镜像源，可通过脚本传入基础镜像：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build_server_image.ps1 `
  -Tag 20260312 `
  -FrontendBaseImage "<your-node-base-image>" `
  -BackendBaseImage "<your-python-base-image>"
```

默认输出：

```text
storage/deploy/algowiki-web-20260312.tar
```

### 4.2 打包项目源码

本地 Windows PowerShell：

```powershell
cd <project-root>
powershell -ExecutionPolicy Bypass -File .\scripts\package_server_source.ps1
```

默认输出：

```text
storage/deploy/algowiki-source.tar.gz
```

源码包会排除：

- `.git`
- `backend/.git`
- `venv`
- `frontend/node_modules`
- `frontend/dist`
- 本地 `.env`
- 运行时日志、媒体文件、静态收集产物

### 4.3 上传到服务器

本地 PowerShell 示例：

```powershell
scp -i $HOME\.ssh\id_ed25519 .\storage\deploy\algowiki-source.tar.gz root@<your-server-ip>:/root/
scp -i $HOME\.ssh\id_ed25519 .\storage\deploy\algowiki-web-20260312.tar root@<your-server-ip>:/root/
```

服务器解压源码：

```bash
mkdir -p /srv/algowiki
tar -xzf /root/algowiki-source.tar.gz -C /srv/algowiki
cd /srv/algowiki
```

## 5. 私有验证部署

当 RDS 还没准备好时，只允许做私有验证，不对公网开放正式站点。

### 5.1 私有验证环境文件

服务器上：

```bash
cd /srv/algowiki
cp deploy/env.private-validation.example deploy/.env.private-validation
```

至少修改这些值：

- `DJANGO_SECRET_KEY`
- `DB_PASSWORD`
- `MYSQL_PASSWORD`
- `MYSQL_ROOT_PASSWORD`
- `SUPERADMIN_PASSWORD`

### 5.2 启动本地 MySQL 容器 + Web 容器

服务器上：

```bash
cd /srv/algowiki
chmod +x deploy/server-compose-up.sh
./deploy/server-compose-up.sh --env-file deploy/.env.private-validation --profile local-db --image-archive /root/algowiki-web-20260312.tar
```

该脚本会：

- `docker load` 导入本地构建好的镜像
- 先执行 `docker compose config`
- 再以 `--no-build` 模式启动容器

注意：

- `APP_BIND` 默认是 `127.0.0.1`
- 这意味着应用只监听服务器本机回环地址
- 如果要从本地电脑访问，建议使用 SSH 端口转发，而不是直接暴露 8001

验证：

```bash
curl -fsS http://127.0.0.1:8001/api/health/
docker compose --env-file deploy/.env.private-validation --profile local-db -f docker-compose.server.yml ps
docker compose --env-file deploy/.env.private-validation --profile local-db -f docker-compose.server.yml logs --tail=100
```

## 6. 正式部署切换

正式部署必须满足：

1. 域名已备案
2. RDS MySQL 已开通
3. RDS 与 ECS 在同地域、同 VPC
4. 数据库只走内网，不开放公网 `3306`

### 6.1 正式环境文件

服务器上：

```bash
cd /srv/algowiki
cp deploy/env.production.example deploy/.env.production
```

至少修改：

- `APP_IMAGE`
- `DJANGO_SECRET_KEY`
- `DJANGO_ALLOWED_HOSTS`
- `DJANGO_CSRF_TRUSTED_ORIGINS`
- `DB_HOST`
- `DB_NAME`
- `DB_USER`
- `DB_PASSWORD`
- `SUPERADMIN_PASSWORD`

### 6.2 切换到 RDS

服务器上：

```bash
cd /srv/algowiki
./deploy/server-compose-up.sh --env-file deploy/.env.production --image-archive /root/algowiki-web-20260312.tar
```

说明：

- 正式环境默认不启用 `local-db` profile
- `docker-compose.server.yml` 中的 `db` 服务只有在 `--profile local-db` 时才会启动
- 正式模式下，应用容器只连接 `DB_HOST=<rds-private-endpoint>`

## 7. 公网开放顺序

推荐顺序：

1. 先完成私有验证
2. 再切 RDS
3. 备案通过后配置域名解析
4. 部署 Nginx 反向代理
5. 再开放公网访问

主机 Nginx 模板位于：

```text
deploy/nginx.algowiki.conf
```

默认建议：

- 容器只绑定 `127.0.0.1:8001`
- 由主机 Nginx 监听 `80/443`
- 公网只暴露 `22/80/443`

## 8. 常见问题

### 8.1 VS Code 提示路径不存在

检查你输入的是不是：

- 正确：`/srv/algowiki`
- 错误：`/root/srv/algowiki`

### 8.2 服务器上无法登录 Codex

如果出现：

- `device code request failed`
- `403 Forbidden`
- `country, region, or territory not supported`

结论就是：不要继续在这台服务器上登录 Codex。改成本地 Codex + Remote-SSH。

### 8.3 Compose 仍尝试构建镜像

请确认：

- 你使用的是 `deploy/server-compose-up.sh`
- 命令里包含 `--image-archive`
- 脚本内部使用的是 `docker compose up -d --no-build`

### 8.4 2G 机器启动时 OOM

优先处理顺序：

1. 开启 swap
2. 确保镜像在本地构建，不在服务器构建
3. 将 `GUNICORN_WORKERS` 保持在 `2`
4. 暂时不要同时跑额外服务
